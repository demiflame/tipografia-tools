# Техническое задание
## Parser v2.0 для AutoHotkey v2

### Цель

Разработать универсальный парсер коммерческих предложений (КП) типографии с двусторонним преобразованием:

- КП → Таблица
- Таблица → КП
- Дополнительные режимы обработки

Проект должен быть модульным, легко расширяемым и не зависеть от одной большой регулярной функции.

---

# Общая архитектура

```
Clipboard
      │
      ▼
 Normalize()
      │
      ▼
 Tokenizer()
      │
      ▼
 Parser()
      │
      ▼
 Formatter()
      │
      ▼
 Clipboard
```

Каждая стадия выполняет только одну задачу.

---

# Горячие клавиши

## Ctrl + Shift + X

Преобразование

**КП → Таблица**

Результат:

```
Наименование    Количество    Цена    Стоимость
```

---

## Ctrl + Shift + C

Преобразование

**Таблица → КП**

Поддерживаются таблицы:

### вариант 1

```
Наименование    Количество    Цена
```

Стоимость рассчитывается автоматически.

---

### вариант 2

```
Наименование    Количество    Цена    Стоимость
```

Используется готовая стоимость.

---

После преобразования автоматически добавляется:

```
Итого к оплате - XXXXX руб

Оплата наличными / счет на Юр. лицо
```

Итог считается автоматически.

---

## Ctrl + Shift + D

Режим диагностики.

Показывает:

- количество строк
- успешно распознано
- не распознано
- причина ошибки

---

## Ctrl + Shift + N

Новый режим.

Преобразует таблицу или КП в краткий список:

```
Наименование - Количество
```

Примеры:

```
Ручка - 50 шт
```

```
Печать наклейки 40х40 мм - 300 шт
```

```
Ролл-ап 200х100 см - 1 шт
```

Используется для быстрого формирования перечня изделий без цен.

# Normalize()

Нормализует текст.

Поддерживает:

- — → -
- – → -
- − → -

Нормализует

```
р/шт
руб/шт
руб / шт
р / шт
```

в единый формат.

Удаляет двойные пробелы.

Не удаляет пробелы между числами.

Например

```
А4 200гр
```

не должно превращаться в

```
А4200гр
```

---

# Tokenizer()

Разбивает строку на логические части.

Не использует сложные регулярные выражения.

Выделяет:

- название
- расчетную часть

Например

```
ПВХ стенд 750х1200мм (7шт) — 1шт х 4800 руб/шт
```

становится

Название

```
ПВХ стенд 750х1200мм (7шт)
```

Расчет

```
1шт х 4800 руб/шт
```

Количество карманов не должно восприниматься как количество изделий.

---

# Parser()

Отдельные функции.

## ParseName()

Извлекает название.

---

## ParseQty()

Определяет количество.

Поддерживает:

```
50шт

50 шт

х 50шт

50шт х

1шт
```

Если количество отсутствует

используется

```
1
```

---

## ParsePrice()

Поддерживает:

```
39 руб

39руб

39 руб/шт

39р/шт

39 р/шт

39
```

---

## ParseTotal()

Если стоимость указана

использует её.

Если нет

вычисляет автоматически

```
Количество × Цена
```

# Код

Проект должен быть модульным.

Рекомендуемая структура:

```
copy_script_v2.ahk

Hotkeys

Normalize()

Tokenizer()

ParseName()

ParseQty()

ParsePrice()

ParseTotal()

ParseKPLine()

KPToTable()

TableToKP()

ShortList()

Debug()

Utils()
```

Каждая функция должна выполнять только одну задачу.

Запрещается создавать одну большую функцию из множества регулярных выражений.

Парсер должен легко расширяться добавлением новых шаблонов без изменения существующей логики.
