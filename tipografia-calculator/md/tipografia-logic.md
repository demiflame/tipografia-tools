# Логика калькулятора типографии

Этот файл — документация к `tipografia-calculator.html`. Не код, просто шпаргалка:
что внутри, как считается цена по каждой категории, как устроены счёт и настройки.
Если что-то в коде поменяется, а здесь не обновится — верь коду, не этому файлу.

---

## 1. Общая архитектура

Один HTML-файл, без сборки и зависимостей. Всё хранится в `localStorage` браузера
под тремя ключами:

| Ключ | Что хранит |
|---|---|
| `tipo_prices_v1` | все цены и тарифы (объект `PRICES`) |
| `tipo_invoice_v1` | текущий счёт (массив позиций) |
| `tipo_company_v1` | реквизиты компании для шапки счёта |

Три вкладки: **Калькулятор** → **Счёт** → **Настройки цен**.

Цены при загрузке = `DEFAULT_PRICES` (зашиты в код) **слитые** с тем, что лежит
в `localStorage` (если юзер что-то поменял в Настройках). Если в коде позже
добавится новое поле цены — у тех, кто уже сохранил свои настройки, оно подтянется
из дефолта автоматически (функция `deepMerge`).

## 2. Как устроен калькулятор (если захочешь добавить категорию)

Все услуги — это один массив `PRODUCTS`. Каждая услуга — объект:

```js
{
  id: "уникальный_id",
  group: "Название группы в меню",
  title: "Что видит пользователь",
  icon: "эмодзи",
  fields: [ ...поля формы... ],
  calc(v, P) { ...формула, возвращает результат... }
}
```

**Поле формы** — это:
```js
{ key:"width", type:"number", label:"Ширина, мм", default:200, min:1,
  showIf: v => v.format==="Свободный" }      // необязательно — когда показывать
```
Для выпадающих списков `type:"select"`, а `options` — либо просто массив,
либо функция `(P) => [...]`, которая берёт варианты прямо из текущих цен
(`P`). Это значит: если в Настройках добавить новую строку цены (например,
новую толщину ПВХ) — она сама появится в выпадающем списке калькулятора,
без правок кода.

`default` тоже может быть функцией `(P) => значение` — так сделано для зазора
в «<10см» у самоклейки: дефолт в форме подтягивается из текущих Настроек.

**`calc(v, P)`** получает значения формы (`v`) и весь объект цен (`P`),
и должен вернуть:
```js
{ total, qty, unitPrice, specs }      // обычный случай
{ error: "текст ошибки" }              // если расчёт невозможен (см. SRA3)
```
`specs` — это готовая строка-описание позиции, которая идёт и на экран
(под итоговой суммой), и в счёт, и в текст для копирования. Один текст — три места.

После `calc()` есть **обязательный шаг округления** (в `computeCurrent()`):
цена за штуку округляется вверх до целого рубля (`Math.ceil`), и итог
пересчитывается от уже округлённой цены (`unitPrice × qty`). Поэтому
в коде самих формул может быть дробная цена — это нормально, округление
происходит снаружи, одним местом для всех категорий.

## 3. Категории и группы

| Группа | Категории |
|---|---|
| 🖨 Печать | Печать SRA3, Визитки, Дипломы/Грамоты, Самоклейка |
| 🧵 Нанесение | ДТФ (DTF), Сублимация |
| 🟦 Жёсткие материалы | ПВХ, Акрил, Холсты |
| 🚩 Большой формат | Баннер, Постер (А2+) |
| 🎁 Сувенирка | Блокноты |

## 4. Формулы по категориям

### Печать SRA3
**Поля:** формат изделия (стандартный из справочника или «Свободный» + свои Ш×В),
плотность, цветность, ламинация (+ мат/глянц), тираж, наценка, скидка.

1. Берём размер изделия (из справочника форматов или вручную).
2. Сколько изделий помещается на физический лист SRA3 (320×450мм, тоже настройка):
   пробуем оба разворота листа, берём максимум.
   Если совсем не помещается — это **ошибка** (`error`), а не ноль.
3. Цена за лист — из таблицы «плотность × цветность» в Справочнике, умножается на 1.8.
4. Делим (цена_листа×1.8 + цена_ламинации) на кол-во изделий на листе → цена за штуку.
5. Скидка за тираж — по общей таблице порогов (см. ниже).
6. Итог = цена_за_шт × скидка_за_тираж × (1+наценка%) × (1−скидка%) × тираж.

**Скидка за тираж** (общая для SRA3, Визиток, Дипломов) — таблица порогов:
от 1 шт — 100%, от 100 — 80%, от 300 — 70%, от 500 — 50%, от 1000 — 40%,
от 10000 — 30% от цены.

### Визитки
**Поля:** плотность, цветность (только для текста — на цену не влияют), тираж.
Цена = фикс. цена за шт × скидка за тираж × тираж. Плотность/цветность в расчёте
не участвуют (так было в исходном Excel, специально оставили).

### Дипломы / Грамоты
Та же логика, что у визиток, но без полей плотности/цветности — просто
фикс. цена × скидка за тираж × тираж.

### ПВХ
Площадь (Ш×В в мм → м²) × цена за м² (по толщине) × коэф. типа резки
× наценка/скидка × тираж.

### Акрил
Площадь = Ширина × (Высота + **Ножка**) — ножка увеличивает площадь и стоимость,
но **не пишется в тексте/буфере** (по твоей просьбе — в расчёте участвует,
в описании скрыта). Дальше × цена за м² (по толщине) × наценка/скидка × тираж.

### Холсты
Площадь (Ш×В в метрах) × цена за м² (зависит от типа: с подрамником / без)
× наценка/скидка × тираж.

### Самоклейка — три формата
- **Свободный**: площадь (мм→м²) × цена за м² × коэф. типа резки.
- **Стикерпак A6 / A5**: **фикс. цена** за штуку (30₽ / 60₽), площадь
  и тип резки не учитываются вообще.
- **«<10см»** — расчёт по площади рулона, см. отдельный раздел ниже.

### «<10см» (мелкие наклейки на самоклейке)
Параметры (настройки, можно менять):
- Ширина рулона — 1300мм
- Вылет с каждой стороны — 100мм → рабочая ширина = 1300 − 100×2 = 1100мм
- Зазор между наклейками — 7мм по умолчанию, **можно переопределить в самой
  форме** под конкретный заказ

Алгоритм:
1. Сколько наклеек шириной W влезает в один ряд:
   `nPerRow = floor((рабочая_ширина + зазор) / (W + зазор))`
2. Сколько рядов нужно под тираж: `rows = ceil(тираж / nPerRow)`
3. Физически вмещается: `fitQty = rows × nPerRow` (может быть больше тиража —
   последний ряд не всегда заполнен полностью)
4. Длина рулона = `rows × (Высота + зазор)`
5. Площадь = **полная ширина рулона** (1300мм, не рабочая!) × длина —
   платим за весь рулон, вылеты — это просто потери ширины, не длины.
6. Стоимость = площадь × цена самоклейки за м².
7. Цена за штуку = стоимость ÷ `fitQty` (физически вместившееся количество,
   а не заказанное — так договорились: чем плотнее заказ ложится в ряд,
   тем меньше цена за единицу).

⚠️ Из этой формулы вытекает: цена за штуку **не зависит от тиража**
(если меняется только количество рядов, а не штук в ряду) — это нормальное
следствие деления на `fitQty`, не баг.

### ДТФ (DTF)
**Поля:** формат (Свободный размер или фикс. формат из справочника),
**материал** (Заказчика / Наш — новое поле), тираж, наценка, скидка.
- Свободный размер: площадь × цена за м².
- Фикс. формат: фикс. цена за штуку.
- Если материал «Наш» — **доплата 750₽ за штуку** (настройка), прибавляется
  до применения наценки/скидки и до умножения на тираж.

### Сублимация
База по типу изделия (А4/А5/Шоппер/Футболка, из справочника) + стоимость
нанесения (площадь печати × та же цена за м², что и у ДТФ-свободного размера) —
всё × наценка/скидка × тираж.

### Баннер
Площадь (Ш×В в метрах) × цена за м² (по плотности) + стоимость отделки
(если выбраны люверсы или проклейка — формула с базовой ценой/делителем/
коэффициентами, все четыре числа — настройки) + Ролл Ап (фикс. доплата,
если выбран) — всё × наценка/скидка × тираж.

### Постер (А2+)
Просто площадь (м) × цена за м² × наценка/скидка × тираж.

### Блокноты
Себестоимость = Обложка + Задник + Пружина + (Лист × кол-во листов в блокноте,
по умолчанию 30) — все четыре цены зависят от формата (А5/А6) и редактируются
как таблица «компонент × формат» в Настройках. Итог = себестоимость × тираж
× наценка/скидка.

## 5. Текст для копирования

Формат одной строки (и в счёте при копировании, и в кнопке «📋 Скопировать»
у одной позиции):

```
<Название и параметры> - <Кол-во>шт х <Цена> руб/шт = <Сумма> руб
```

Числа форматируются «по-умному» (`fmtSmart`): без лишних нулей, целое — без
дробной части (`200`, не `200,00`), запятая вместо точки. Поскольку округление
до целого рубля уже происходит раньше (см. раздел 2), на практике в строке
почти всегда целые числа.

Полный текст счёта (кнопка «📋 Скопировать счёт»):
```
[Название компании]         ← если заполнено в Настройках
[Телефон]                   ← если заполнено
[Доп. строка]                ← если заполнено

<строка позиции 1>
<строка позиции 2>
...

Итого: <сумма> рублей

Оплата: <текст из настроек, по умолчанию «Наличными / Счёт на Юр. лицо»>
```

## 6. Счёт

- Позиции добавляются из калькулятора («➕ Добавить в счёт») или вручную
  через форму «+ Своя позиция» (название, кол-во, цена).
- В таблице счёта можно прямо на месте поменять **количество** и **цену**
  у любой позиции — пересчёт суммы происходит сразу. Цена при ручном вводе
  тоже округляется вверх до целого рубля.
- На телефоне таблица превращается в карточки (по одной на позицию),
  на широком экране — обычная таблица.
- Хранится в `localStorage`, переживает обновление страницы.

## 7. Настройки

Каждый блок настроек — это просто редактор поверх объекта `PRICES`. Изменение
в любом поле сразу сохраняется в `localStorage` и сразу пересчитывает
калькулятор. Есть три вида редакторов:
- **простое число** (например, цена за м²)
- **список «ключ → цена»** с кнопками добавить/удалить строку
- **таблица «строка × столбец»** (например, плотность × цветность у SRA3,
  компоненты × формат у блокнотов) — можно добавлять и строки, и столбцы

Кнопки сверху: экспорт/импорт настроек в JSON-файл (для бэкапа) и сброс
к заводским ценам.

### ⚠️ Неиспользуемые остатки данных
В объекте `DEFAULT_PRICES` остались блоки `engraving`, `flagsExtra`,
`bracelets`, `badges`, `pvdBags` — это цены для категорий «Гравировка»,
«Флаги/виндеры», «Браслеты», «Значки», «ПВД пакеты», которые убрали из
калькулятора и из Настроек. Сами данные не удалял — мало ли, вернёте
обратно — но сейчас они нигде не читаются и не показываются.

## 8. Известные особенности (не баги)

- Цена за штуку у формата «<10см» не зависит от количества рядов с одинаковой
  заполненностью — см. раздел про самоклейку выше.
- У Визиток поля плотности/цветности не влияют на цену — это сознательное
  решение (так было в исходном Excel-файле).
- Скидка за тираж (таблица порогов) общая для трёх категорий: SRA3, Визитки,
  Дипломы. У остальных категорий своих «скидок за объём» нет — только ручные
  поля «Наценка %» / «Скидка %».
